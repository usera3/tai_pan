# 文件下载与删除设计

## 目标

在现有本地钛盘管理器的文件列表中增加下载和删除操作，仅使用钛盘公开、稳定的 API Key 接口。文件夹创建、删除和移动不在本次范围内，因为官方 API Key 接口未开放这些能力。

## 下载

- 文件行提供下载图标按钮。
- 本地后端通过 `link_add` 为目标 UKEY 创建临时直链。
- 直链有效期为 1440 分钟（24 小时）。
- 不设置 `download_limit`，以支持大文件、断点续传和多线程下载。
- 前端请求本地后端成功后，通过当前页面内的隐藏 iframe 触发下载，管理页面和当前视图保持不变。
- 创建失败时不创建 iframe，并在当前页面显示服务端错误。
- 临时直链到期后自动失效，源文件不受影响。
- 自动下载直链写入被 Git 忽略的 `.local/download_links.json` 注册表。
- 同一 UKEY 已有至少 1 小时剩余有效期的下载直链时直接复用，不重复创建。
- 直链列表过滤注册表中的 DKEY；用户通过“创建直链”主动创建的记录不进入注册表，因此正常显示。
- 注册表保留历史自动 DKEY，使尚未从远端消失的下载直链始终保持隐藏；过期记录不再用于下载。

## 删除

- 文件行提供危险样式的删除图标按钮。
- 删除弹窗显示文件名，并明确提示源文件及其全部直链都会失效。
- 用户确认后，本地后端先调用 `link_add` 为 UKEY 创建临时直链并取得 DKEY，再调用 `link_del`，同时传入删除源文件参数。
- 只有第二步成功后才向前端报告删除成功。
- 删除成功后刷新文件列表、直链列表和概览数据。
- 任一步失败都保留真实错误提示，不报告成功。

## 后端接口

- `POST /api/files/{ukey}/download`：创建 24 小时直链，返回标准化下载 URL 所需的远端数据。
- `DELETE /api/files/{ukey}`：执行“创建临时直链，再删除直链和源文件”的组合操作。
- DKEY 提取兼容远端返回字符串或对象；无法取得 DKEY 时返回明确业务错误，禁止继续删除。

## 安全与边界

- API Key 继续只保存在本地设置文件中，不返回浏览器。
- 下载直链注册表只保存 UKEY、DKEY、下载 URL 和到期时间，不保存 API Key。
- UKEY 和 DKEY 作为路径参数时进行正常 URL 编码和校验。
- 不调用钛盘网站内部登录接口，不保存登录 Token。
- 不伪造文件夹功能。
- 删除不可撤销，必须由用户在弹窗中明确确认。

## 测试

- 客户端测试验证下载直链使用 `valid_time=1440` 且不发送 `download_limit`。
- 路由测试验证下载返回、组合删除顺序、DKEY 提取及中间失败行为。
- 前端契约和 Playwright 测试验证下载按钮、删除确认弹窗、成功刷新和错误提示。
- 完整运行 Python 与 Chromium 测试，并在真实本地服务上验证非破坏性的下载链接创建。
